使用pageHelper用于简化分页查询操作

添加依赖

正常来说，我们编写分页的查询的sql需要用到limit语句，并且需要根据页码与每页的数据量计算查询的范围

LIMIT offset, row_count

  • offset ：表示跳过的记录数，从 0 开始，一般通过计算得出

  • row_count ：表示返回的最大记录数

pageHelper通过拦截MyBatis的SQL执行，自动在查询语句中添加分页 SQL，从而实现分页功能

现在不用手写分页查询语句了，但是要注意，如果使用pageHelper，那么数组访问层返回值需要使用Page类型接收

Page

分页结果类，封装了分页数据和分页元数据，方便在业务层和控制层中使用

一些常用方法

  • List getResult() ：获取分页查询的结果列表

  • int getTotal() ：获取总记录数

  • int getPages() ：获取总页数

估计是pageHelper要操作这一种类型的数据

查完封装成pageResult返回即可

法二：扩展SpringMVC消息转换器

SpringMVC消息转换器：http请求与Java对象间的互换操作

MappingJackson2HttpMessageConverter：消息转换器中的一种

SpringMVC内置了好多种转换器，这是其中的几种

  • MappingJackson2HttpMessageConverter 用于处理 JSON 数据（基于 Jackson 库）

  • Jaxb2RootElementHttpMessageConverter 用于处理 XML 数据（基于 JAXB）

  • StringHttpMessageConverter 用于处理纯文本数据

  • FormHttpMessageConverter 用于处理表单数据（ application/x-www-form-urlencoded ）

  • ResourceHttpMessageConverter 用于处理文件资源（如下载文件）
