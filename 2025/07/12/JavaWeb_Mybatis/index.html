Mybatis

  • 扫描注解MapperScan：Mapper注解想要生效，同样需要一个扫描注解MapperScan，作用同Spring提供的ComponentScan注解（用于Bean组件的扫描）

    有趣的是，ComponentScan也会去扫Mapper，而ComponentScan就在启动类的SpringBootApplication注解中，因此只要在标准Spring项目中，不用显式添加MapperScan也能扫到Mapper

JDBC

(Java DataBase Connectivity)，就是使用Java语言操作关系型数据库的一套API，java程序中需要使用这套API与数据库进行交互，而Mysql、Oracle等数据库产品实际上就是封装了这套Api。我们在java程序中连接各种数据库时，表面上语法一致（JDBC接口），底层使用的其实是不同厂商的实现，Mysql提供Mysql实现，Oracle提供Oracle的实现。实际上JDBC提供的只是一个标准，一套接口——内部没有具体实现，不同产品根据这套标准完成了自己的API——JDBC实现，也称数据库驱动

  • sun公司官方定义的一套操作所有关系型数据库的规范，即接口

  • 各个数据库厂商去实现这套接口，提供数据库驱动jar包。

  • 我们可以使用这套接口(JDBC)编程，真正执行的代码是驱动jar包中的实现类

直接使用JDBC如何与数据库交互，大致分4步

  • 注册驱动：JDBC只是个标准，需要提供数据库驱动才能使用

  • 获取连接：填写数据库信息

  • 获取执行SQL的对象Statement，执行SQL：请求并获取数据，解析数据

  • 释放资源：关闭与数据库的连接

再来看看Mybatis是怎么完成这些步骤的

  • 在配置信息中完成注册驱动、获取连接；

这样做能避免在java程序中硬编码数据库信息，因为数据库是需要经常改变的，所以提取到外部更方便改动

  • JDBC中获取的数据是一个ResultSet类，需要手动解析内部数据，还要手动封装到集合，非常麻烦；而Mybatis返回的对象是我们编写的对象，不用解析，并可以直接封装到集合中

  • JDBC中每一次询问都要先建立连接，最后释放连接，频繁操作会造成资源浪费；Mybatis在配置数据库信息时使用了(spring.datasource.)接入了spring底层的数据库连接池，数据库连接池会统一管理和分配这些连接（有点像ioc容器，但主要目的不同），也就是说，当程序与数据库A建立连接时，连接池会给程序分配与A的连接，程序完成交互，连接池再回收连接，使用数据库连接池的好处如下：

    1. 减少连接开销：避免每次数据库交互都进行连接创建和销毁，从而减少系统资源消耗和时间延迟
    2. 提高性能：通过重用连接，减少了连接创建的时间，提高了应用的响应速度
    3. 连接限制：控制同时打开的数据库连接数量，避免过多的并发连接导致数据库服务器过载
    4. 故障恢复：实现连接的自动重连和失效连接的剔除，保证连接的稳定性

数据库连接池

  • 资源重用

    数据库连接池是个容器，负责分配、管理数据库连接(Connection)

  • 提升系统响应速度

    它允许应用程序重复使用一个现有的数据库连接，而不是再重新建立一个

  • 避免数据库连接遗漏

    释放空闲时间超过最大空闲时间的连接，来避免因为没有释放连接而引起的数据库连接遗漏

预编译sql

这种携带#{ }的sql语句称为预编译sql，采用预编译sql有两大优势：

1.性能高

不只是增加了代码的复用性，这里的性能与sql底层的编译机制有关

sql语句的执行流程

  • SQL语法解析检查

  • 优化SQL

  • 编译SQL

  • 执行SQL

java向数据库服务器发送SQL语句后，服务器需要对SQL语句进行处理才会执行，处理分为语法检查，优化，编译这三步，而优化编译后的语句会被缓存起来，当接收到编译过的sql语句时，就不必进行处理，可以直接执行缓存过的语句

如果不使用预编译方式，而是直接写入数据，删除不同id的语句会被多次编译

如果使用预编译，#{}在编译后会被替换为?，因此这三条语句编译后其实是一样的，执行阶段？将被替换为原来的参数值，这样就只编译了一次，提高了性能

2.更安全

用于防止sql注入

sql注入：通过操作输入的数据来修改事先定义好的SQL语句，以达到执行代码对服务器进行攻击的方法

Results&Result

在MyBatis中，Results和Result注解用于映射SQL查询结果到Java对象的属性。这在处理复杂查询或当数据库列名与Java对象属性名不完全匹配时非常有用

Results：Results注解是一个容器，用于包含多个@Result注解。它定义了一组结果映射规则，这些规则将数据库中的列映射到Java对象的属性

Result：Result注解用于定义单个映射规则，指定数据库中的列名和Java对象的属性名之间的映射关系。它通常包含以下属性：

  • column: 数据库中的列名
  • property: Java对象的属性名
  • javaType: 属性的Java类型（可选）
  • jdbcType: 属性的JDBC类型（可选）
  • id: 标记这个映射是否是结果集的主键（可选）
