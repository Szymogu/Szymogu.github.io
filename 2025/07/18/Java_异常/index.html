异常

java中的异常体系

爷类🥸：Throwable

父类：Error、Exception

Error：系统级别错误，java内部问题，可以不用管

Exception：异常，程序可能出现的问题

Exception的子类：RuntimeException、其他

编译时异常：除了RuntimeException与其子类，其他都是编译时异常

运行时异常：RuntimeException本身与其子类

异常的作用

两种

1.bug的关键信息

2.方法的特殊返回值，可用于通知调用者它的执行情况

异常处理方式

三种

1.JVM默认处理方式

2.自己处理

3.抛出异常

JVM默认处理方式

红色字体输出异常信息，中断程序执行

自己处理

又称捕获异常

使用try-catch结构

try：可能出现异常的代码

catch(异常类型 变量名)：可处理异常

异常被捕获后，程序不会终止，可以继续执行后面的代码

关于try-catch的细节:四个问题

一：如果try中没有遇到问题，怎么执行?

不执行catch
二：如果try中可能会遇到多个问题，怎么执行，如何正确捕获?

遇到第一个异常时停止try，执行对应catch

使用多个catch捕获不同类型的异常，且异常存在父子关系的，父类异常写在后面
三：如果try中遇到的问题没有被捕获，怎么执行?

JVM处理异常
四：如果try中遇到了问题，那么try下面的其他代码还会执行吗？

不执行

异常中的常见方法

public String getMessage()：返回此throwable的消息

public String toString()：返回throwable的类型+信息

public void printStackTrace()：输出异常（红色），但是不会中断程序

执行结果

throw&throws

throw：抛出一个异常对象，中断该方法的执行

throws：声明方法中可能抛出的异常（非运行时异常一定要声明）

外层方法要记得处理这些异常：可以继续抛出，也可以直接捕获处理

自定义异常

让异常更加明确

四步骤

1.定义异常类

异常类名命名规范：异常名+Exception

2.写继承关系

运行时异常：继承RuntimeException

编译时异常：继承Exception

3、4.空参构造、带参构造（错误信息）
