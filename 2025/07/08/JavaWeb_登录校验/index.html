校验jwt

首先思考，为什么要校验，校验的是什么？

为什么要校验，这个问题换一种方式问就是

为什么我们不直接使用json作为校验码？

如果有中间者对json数据进行篡改，服务器完全无法知道，这不安全

jwt做了什么？它利用签名校验防止数据被篡改，而生成签名的密钥只有服务才知道，因此中间者无法神秘地修改内容而不被发现。

因此我们也能得知，校验的是数据完整性（是否被篡改）

所以校验就两步

密钥+jwt校验码

Filter

过滤器，用于统一处理请求，是Javaweb三大核心组件之一

• Servlet(小服务程序）：用于处理 HTTP 请求和响应，是 Java Web 应用的核心组件

• Filter（过滤器）：用于过滤请求和响应，可以对请求进行预处理或后处理

• Listener（监听器）：用于监听应用生命周期中的各种事件，可以管理资源或记录事件

考虑以下场景，如果没有拦截，在未登录状态下，我们可以使用路径直接访问一个资源，这不是我们想要的，因此对于每一个资源访问请求，我们都需要校验这次访问的合法性（使用jwt），但这么多请求处理方法上，加的是同样的校验代码，能不能把校验的事情抽取出来让一个模块去完成呢？这就是filter的作用：统一预/后处理请求

预处理：校验合法性

后处理：返回正确资源/拒绝访问

使用方式

实现接口Filter，这个接口有三个方法：

  • init：初始化Filter，只执行一次。提供了默认配置，可以不用实现

  • doFilter：主要逻辑，用于过滤请求

  • destrory：释放资源，仅执行一次

实现方法后，为类添加webFilter注解，表明这是一个过滤器类，后面添加拦截路径

通过在doFilter中调用doFilter方法放行本次请求

filterChain.doFilter(servletRequest,servletResponse);

执行流程

进入方法后，首先调用init初始化Filter，随后进入doFilter执行逻辑，当调用dofilter时，会放行这个请求，随后执行后面的逻辑，最后调用destroy释放资源

拦截路径

可以用逗号分隔多个需拦截路径

过滤器链

当存在多个过滤器时，请求到来时容器会构建一条过滤器链，只有那些会拦截这个请求路径的过滤器会被加入到过滤器链中

经过一个过滤器时，有两种情况：

1.通过，过滤器调用doFilter将该请求传递到下一过滤器上

2.不通过，直接返回响应

当链上的最后一个过滤器通过该请求，此时会倒过来逐一执行链上过滤器剩余部分的代码，类似一个栈

执行流程

一个请求来到服务器（假设是tomcat），首先会触发服务器的Filter，放行后进入spring模块，回忆一下，tomcat对在它上面运行的程序是有要求的——需实现servlet接口。tomcat通过spring的接口实现类——DispatcherServlet与spring中的内容交互，也就是说，服务器不能直接与spring中的controller交互，而是要借助DispatcherServlet，此时我们又添加了拦截器（intercepter），请求会先经过拦截器，通过才能到达controller

之前实现filter时有这么一段代码

请解释：httpServletRequest.getRequestURL()

分析一下，httpServletRequest是一个HttpServletRequest类型的对象，而HttpServletRequest是一个接口，接口肯定是不能直接调用方法的，因此只有一种可能——多态，servletRequest是一个接口的实现类。的确，是tomcat提供了ServletRequest接口的具体实现类RequestFacade，因此这里servletRequest的实际类型是RequestFacade，在调用HttpServletRequest的getRequestURL方法时，是调用了RequestFacade中的getRequestURL

使用方式

两个步骤：1、创建拦截器 2、注册拦截器

创建拦截器

拦截器需要实现HandlerInterceptor接口，这个接口中有三个需实现的方法（都有默认）

  • preHandle 在请求处理之前进行调用

  • postHandle 请求处理之后，但在视图被渲染之前进行调用

  • afterCompletion 在视图渲染完成后执行

完成拦截工作的是prehandle，因此先实现preHandle即可，它有两个返回值

  • true 通过，不拦截

  • false 未通过，拦截

注册拦截器

让拦截器生效

新建一个类实现WebMvcConfigurer接口并使用@Configuration标记为配置类

重写addIntercepter方法，形参registry是一个工具类，用于注册拦截器

调用registry中的方法addInterceptor()，传入需要添加的拦截器，会把这个拦截器添加到拦截器链上，返回值为新添加的拦截器

调用addPathPatterns()，可设置需拦截的路径

调用excludePathPatterns()，可设置需排除拦截的路径
