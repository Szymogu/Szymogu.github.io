作业地址

ostep-homework/cpu-intro at master · remzi-arpacidusseau/ostep-homework

使用命令运行程序

python process-run.py -h

得到结果如下

Usage: process-run.py [options]

Options:
-h, –help show this help message and exit
-s SEED, –seed=SEED the random seed
-l PROCESS_LIST, –processlist=PROCESS_LIST
a comma-separated list of processes to run, in the
form X1:Y1,X2:Y2,… where X is the number of
instructions that process should run, and Y the
chances (from 0 to 100) that an instruction will use
the CPU or issue an IO
-L IO_LENGTH, –iolength=IO_LENGTH
how long an IO takes
-S PROCESS_SWITCH_BEHAVIOR, –switch=PROCESS_SWITCH_BEHAVIOR
when to switch between processes: SWITCH_ON_IO,
SWITCH_ON_END
-I IO_DONE_BEHAVIOR, –iodone=IO_DONE_BEHAVIOR
type of behavior when IO ends: IO_RUN_LATER,
IO_RUN_IMMEDIATE
-c compute answers for me
-p, –printstats print statistics at end; only useful with -c flag
(otherwise stats are not printed)

  • -h 显示此帮助消息并退出

  • -s 随机种子

  • -l 要运行的进程列表，以逗号分隔，格式为X1:Y1，X2:Y2，…其中X是进程应该运行的指令数，Y是一条指令使用CPU或发出IO的概率（从0到100）

  • -L 单次IO所需单位时间

  • -S 是否允许进程在IO阻塞时切换到另一进程：SWITCH_ON_IO， SWITCH_ON_END

  • -I IO结束时的行为类型：IO_RUN_LATER（等其他进程完成）， IO_RUN_IMMEDIATE（CPU资源下次释放时立刻回到我）

  • -c 为我计算答案

  • -p 最后打印统计信息；只对-c标志有用（否则不打印统计信息）

使用案例

1.python process-run.py -l 5:100 -c

运行含一个进程的列表，这个进程含5条指令，每条指令是 CPU 指令的几率是 100%.

-c 查看详情如下

Time PID: 0 CPU IOs
1 RUN:cpu 1
2 RUN:cpu 1
3 RUN:cpu 1
4 RUN:cpu 1
5 RUN:cpu 1

单个进程PID：0，运行了5条cpu指令

2.python process-run.py -l 5:100,5:100

运行两个进程

-c 查看详情如下

Time PID: 0 PID: 1 CPU IOs
1 RUN:cpu READY 1
2 RUN:cpu READY 1
3 RUN:cpu READY 1
4 RUN:cpu READY 1
5 RUN:cpu READY 1
6 DONE RUN:cpu 1
7 DONE RUN:cpu 1
8 DONE RUN:cpu 1
9 DONE RUN:cpu 1
10 DONE RUN:cpu 1

这里出现了多种进程状态，在进程1使用CPU时（RUN:cpu），进程2为（READY），进程1运行完成（DONE）时，进程2开始运行（RUN:cpu）

3.python process-run.py -l 3:0 -L 5

现在加入IO事件，单个进程，三条指令，均为IO，单次IO需要5单位时间

运行结果显示三个IO请求轮流运行

4.python process-run.py -l 3:0 -L 5 -c

程序发出三次IO，而每次发出IO进程进入阻塞（BLOCKED），观察到此时CPU是空闲的（只有在执行指令时，使用CPU）

便于理解，这里的IO启动与完成所用的指令数为1
