集合进阶（上）

java中有两类集合，它们应两个顶层接口

Collection：单列集合

Map：双列集合

Collection

单列集合顶层接口

常用方法

add()：增

clear()：清空

remove()：删给定元素

contains()：集合是否包含给定元素

isEmpty()：集合是否为空

size()：集合元素数量

遍历方式

  • 迭代器：遍历过程中可删除元素

  • 增强for：简化迭代器使用，底层同样使用迭代器，遍历过程中不能删除元素

  • Lambda（foreach）：遍历过程中不能删除元素

Iterator

迭代器，集合的通用遍历方案，不依赖索引

常用方法

List.iterator()：获取一个集合迭代器对象

hasNext()：迭代器当前位置是否有元素

next()：获取当前元素并后移迭代器

细节

  • 每次读取元素前，先检验当前位置是否有元素，没有元素时还读取会抛异常

  • 迭代器遍历后，不会复原

  • 迭代器遍历中，不能使用集合提供的方法添加、删除元素（并发问题），删除元素可以使用迭代器提供的方法

List

列表

常用方法

add(index, element)：在集合指定位置插入指定元素

remove()：删除元素，有两个重载方法：形参传入int类型表下标，传入Object类型表对应元素

set(index, element)：修改指定位置的元素

get()：返回指定下标的元素

遍历方式

列表迭代器：可添加元素

结果：

aaa
bbb
ddd
ccc

ArrayList

空参创建集合时，底层创建一个默认长度为0的数组

调用add方法首次添加元素时，底层默认创建一个长度为10的数组（大于10根据实际需求创建）

存满时默认扩容1.5倍（1.5倍扩容后还存不下，则根据实际需求扩容）

详见之前的笔记

LinkedList

双向链表

常用方法

addFirst()：头插

addLast()：尾插

getFirst()：获取头

getLast()：获取尾

removeFirst()：删除头

removeLast()：删除尾

泛型进阶

泛型通配符

解决泛型的不可继承问题

由于泛型是不能继承的，意味着 List 和List是完全不相关的两个类型

假设现在还有一个List，请问我该怎么声明一个仅可以遍历List类型和List类型的方法

试试多态？

怎么报错了：提示print方法只接受List类型，这就是上面提到的“泛型不可继承问题”，List不是List的子类

这时候就要用到泛型通配符了：它一般就是用来做“容器的多态”的

结论先行

？

两个写法

? extends E：接受E与其子类

? super E：接受E与其父类

因此上面的意思是：可接收Animal与它的子类

我在举例时，为了方便写了一段这样的代码，发现它报错了

‘fanxing. test. this’ cannot be referenced from a static context

意思是：静态方法中不能实例化非静态内部类

为什么呢？

这涉及到内部类的一些机制：编译时，会为内部类添加其所属实例的引用

而当外部类没有实例时，内部类自然无法成功创建。上面的报错就是这个原因，main方法使用static修饰，也就是说它不依赖与实例存在，没有Test类的实例，它的内部类也就无法成功实例化了

既然关键在于Test类的实例，我创一个是不是就行了？确实

Set

集合

无序、不重复、无索引

“适合去重的数据结构”

Hash

哈希值，int类型，对象对应的一个值。默认是Object中实现的，使用地址值计算得到，通常不同对象不会有相同的值；少数情况相同，即哈希冲突问题

HashSet

一个默认长度为16的数组，当存储元素超过数组容量的0.75倍时，会扩一倍

根据元素的哈希值计算他应该存在数组的什么位置：（数组容量-1）&哈希值

不重复的元素会添加在数组的那个位置，并形成一个链表

当链表长度大于8，且数组长度大于等于64时，链表会转换为红黑树

特点：

  • 存取顺序不同

    存的位置是靠哈希值计算，而取用迭代器遍历数组与链表

  • 去重的手段

    hashcode+equals 一般的使用都要重写这两方法，以达到用对象属性去重的目的（而不是靠地址去重）

  • 没有索引

    链表、红黑树这样的复杂结构混合，不适合用索引表示

LinkedHashSet

节点间多了一个双向链表，用于存储存入顺序

遍历时使用这个双向链表（而不是HashSet那样依次遍历数组上的结构），以达到存取有序

多一个链表造成了额外开销，因此没有要求存取的顺序相同时，最好用HashSet去重

TreeSet

不重复、无索引、可排序

底层使用红黑树实现排序，有较好的增删改查性能

排序时会使用类的compareTo方法，如果类需要排序，需要实现Comparable接口

compareTo方法跟之前sort方法自定义排序规则时，传入的Comparator实现类中的方法类似，根据返回值确定当前元素排前还是排后

当返回值为负：排左

当返回值为正：排右

*当返回值为零：表示当前元素重复，舍弃该元素

由于TreeSet是红黑树结构，因此排左、排右指的是左子or右子，排序完成后，检查结构是否符合红黑规则并统一调整

Cat{name=’mimi’, age=1}
Cat{name=’cici’, age=2}
Cat{name=’haha’, age=3}

Animal类

神奇的@SuperBuilder注解

Lombok提供的@AllArgsConstructor和@NoArgsConstructor都不会主动调用父类的构造方法，这对继承体系中的子类不太友好，一般的解决办法是子类手写构造方法

但也可以用Lombok提供的SuperBuilder注解

记得在父子上都要加，使用方式跟Builder类似，实际上SuperBuilder就是Builder+继承功能，可以让子类访问父类字段

还有一种定义排序的方式是传入一个比较器

假设说我现在像根据字符串的长度排序，但String默认实现的compareTo方法是根据字典序怎么办

lambda神力，这里的用法同sort——实现Comparator接口
