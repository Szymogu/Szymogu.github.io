Ajax&Axios

Created2024-07-27|Updated2024-08-22|前端Ajax
|Post Views:

Ajax

概念

异步JS与XML（Asynchronous JavaScript And Xml）

作用

数据交换：通过Ajax可以给服务器发送请求，并获取服务器响应的数据。

通过Ajax动态发送请求,服务器返回数据，html再基于Vue指令渲染数据到页面。

异步交互：在不重新加载整个页面的情况下，与服务器交换数据并更新部分网页。

同步与异步

同步：用户端发送请求后，需等待服务器响应才能进行下一次操作。例如在访问一个网址时，浏览器向服务器发送了一个请求，需要等服务器响应并返回数据后，网页才能成功渲染。

异步：用户端发送请求后，无需等待服务器响应，依然可以操作页面。比如在浏览器搜索框输入内容时，浏览器会根据内容进行搜索联想，并且在改变输入内容时，联想内容也会动态的改变，这就是异步交互。

原生Ajax

基本流程

分三步：

1.创建XMLHttpRequest(XHR)对象

用于与服务器进行数据交换

在抓包工具中XHR选项代表异步请求

2.调用xmlHttpRequest中的open方法，并调用send方法

open方法用于设置请求方式以及请求路径

sent方法用于发送异步请求

3.监测服务器状态是否发生变化，使用xmlHttpRequest的onreadystatechange方法，当状态改变时会触发函数。这个函数有两个作用，一是判断是否成功获取数据，二是渲染数据

尝试获取服务器的响应数据并渲染

例：

关于如何判断是否成功获取数据，这里使用的是

readyState：保存XMLHttpRequest的状态

0：请求未初始化

1：服务器连接已建立

2：请求已收到

3：正在处理请求

4：请求已完成且响应已就绪

status：返回请求的状态号

200：”ok”

403：”forbidden”

404：”not found”

Get&Post

两种最常见的Http请求方式

  1. GET请求：

    • 用于请求服务器发送资源。
    • 通常用于查询操作，不会改变服务器上的资源状态。
    • 参数通过URL传递，附加在URL后面，形成查询字符串。
    • 可以被缓存、收藏为书签、保留在浏览器历史记录中。
    • 对数据长度有限制，因为URL长度有限制。
    • 因为参数暴露在URL中，不适合传输敏感信息。
    • 应该是幂等的，即多次执行相同的GET请求，资源的状态不会改变。

  2. POST请求：

    • 用于向服务器提交数据进行处理，比如表单提交。
    • 通常用于创建或修改服务器上的资源。
    • 参数包含在请求体中，不在URL中。
    • 不会被缓存，也不会保存在浏览器历史记录中，无法收藏为书签。
    • 对数据长度没有特别的限制，取决于服务器的配置。
    • 适合传输大量数据和敏感信息，因为数据不会显示在URL中。
    • 不是幂等的，多次执行相同的POST请求可能会每次都改变资源状态。

综合练习（Vue+Axios）

需求

页面加载完成后，自动发送异步请求，加载数据，渲染展示页面。

做法

通过Vue自动触发钩子函数调用Axios的get请求以异步请求页面数据，将返回值存入Vue的数据模型，再由Vue选择渲染区间和渲染元素，从而达成页面的动态渲染

由于练习用到的服务器接口失效，所以还是请出我们的老朋友。

难点

this.：使用data中的emps存储了返回的部分数据（不需要管这个路径是什么，会返回一个具有三个元素的数组），需要使用this.表示这个Vue对象的emps。

table：在使用tr&td标签循环渲染数据时，没有添加table，导致渲染失败，tr 元素不能直接作为 table 的子元素，它们应该放在 tbody 或 thead 中。
