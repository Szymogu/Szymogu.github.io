集合

Collection

所有单列集合祖宗接口

底下分为两类：List、Set

List

添加的元素有序、可重复、有索引

  • 有序：先进先出，类似c++的队列

  • 可重复：数据值可重复

  • 有索引：可通过索引访问元素

Set

添加的元素无序、不重复、无索引

常用方法

  • public boolean add(E e)

    将给定对象添加到当前集合

    对于List类型集合，不会添加失败；而Set类型集合由于不重复特性，在添加一个已有元素时，会返回False

  • public void clear()

    清空集合

  • public boolean remove(E e)

    移除指定对象

    由于该方法属于Collection，Collection下的Set类型无索引，因此不能使用下标删除元素

  • public boolean contains(Object obj)

    判断当前集合是否包含指定对象

    如果集合中存储的是自定义对象，需要在类中重写equals方法：contains底层使用equals遍历判断集合中的元素是否等于给定元素，这里调用的是给定元素的equals方法，如果给定元素没有重写equals方法，就会使用其父类Object的equals方法：使用==比较地址值，而我们多数时候希望比较值而不是地址，以至于会得到错误的结果

    *此外，一些java类已经重写了equals方法，如String：底层判断值是否相等。这些类就不用自己重写equals了

  • public boolean isEmpty()

    判断集合是否为空

  • public int size()

    返回元素个数

遍历

集合有三种遍历方式

  • 迭代器遍历

  • 增强for

  • Lambda表达式

迭代器遍历

使用迭代器获取集合中的元素

迭代器：类似于指针

例：使用迭代器遍历集合并打印

hasNext：当前位置是否存在元素

next：返回当前元素并将迭代器向后移动1位

more details

1.迭代器指到空时获取元素报错NoSuchElementException，而不是ArrayIndexOutOfBoundsException

2.迭代器遍历完毕，指针不会复位

3.循环时注意next与hasNext的配合使用

4.迭代器遍历时，不能使用集合的方法进行添加或删除

增强for

底层就是迭代器，单列集合、数组可以使用

for(类型 变量名 : 集合/数组)

例：使用增强for遍历集合

List

是一个接口，它是Collection接口的一个子接口，提供了对有序集合的操作

由于List集合有索引，所有天骄了很多索引操作的方法

常用方法

  • void add(int index,E element)

        在此集合中的指定位置插入指定的元素

  • E remove(int index)

    删除指定索引出的元素，返回被删除的元素

  • E set(int index,E element)

    修改指定索引处的元素，返回被修改的元素

  • E get(int index)

    返回指定索引处的元素

detail

add:

在插入元素时，add的底层通过arraycopy将插入位后的元素复制到原数组中（元素后移），再将新元素加入数组

remove:

下列代码执行的是什么操作？

A.删除索引为0的元素

B.删除值为0的元素

答案是A

在调用方法时，如果方法出现了重载现象，会优先调用实参与形参一致的方法

A选项对应的方法是remove(int index);

B选项对应的方法是remove(Object o);

此时提供的参数是0，也就是基本数据类型int，虽然它自动装箱后可以变成Integer，但A选项的优先级更高，因此这种情况会直接当成索引处理，而不会进行装箱

List下三种结构的底层分析：

ArrayList

底层数据结构：数组

  • 利用空参创建的集合，在底层创建一个默认长度为0的数组

  • 添加第一个元素时，底层会创建一个新的长度为10的数组

  • 存满时，会扩容1.5倍

  • 如果一次添加多个元素，1.5倍还放不下，则新创建数组的长度以实际为准

LinkedList

底层数据结构：双向链表

  • 内部类Node，表示节点

Iterator

迭代器（你怎么进来的）

  • 调用迭代器构造方法时，会创建内部类Itr对象，这个对象里面的属性有

    1.cursor：当前索引

    2.lastRet：上一次操作的索引

    3.expectedModCount：修改次数

  • 通过next()方法遍历集合时，先检查是否存在并发修改异常：expectedModCount==modCount？不相等时会抛出异常

    通过cursor获取当前位置，做一些判断后，将当前位置赋值给lastRet并返回当前位置的值

Vector

底层数据结构：数组

类似于ArrayList，但Vector是同步的，这意味着它是线程安全的。Vector是Java早期版本中的一个类，随着Java集合框架的发展，Vector逐渐被ArrayList和LinkedList所取代
