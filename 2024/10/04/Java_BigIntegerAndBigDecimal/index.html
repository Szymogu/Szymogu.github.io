BigInteger&BigDecimal

BigInteger

大整数

构造方法：

Biginteger(int numBits,Random rnd)：获取随机最大整数，范围为[0,2^numBits-1]

Biginteger(String val)：获取指定大整数，注意这里的String要是一个整数，浮点数与其他字符都会报错

Biginteger(String val, int radix)：将指定进制数字转换为大数。【字符串中的数字必须是整数，且要与给定的进制吻合】

静态方法

valueOf()：获取一个整数并转换为大整数。【这个方法的形参类型long，

更多细节：与构造方法BigInterger(String val)的区别

  • 静态方法能处理的数据范围为long

    构造方法处理数据范围更大

  • 静态方法在其内部会提前创建-16~16的BigInteger对象，多次调用时不会创建新对象。

例子：创建两组对象，一组小于等于16，另一组大于16，比较其地址是否相同(==)

输出结果为：1

认为当传入值为-16~16范围内时，会返回先前创建好的对象，因此1，2的地址值相同。但超出这个范围的数值，就会重新创建这个值的BigInteger对象，因此3，4的地址值不同。

  • 对象创建后，内部的值不能改变。

  • 进行计算时会产生一个新的BigInteger对象

这两点表明了BigInteger在进行运算时，不会改变自身的值，而是创建并返回一个值为计算结果的BigInteger对象。【在.add()方法中，若a.add(b)，而a或b的值为0，此时不会创建对象，而是直接返回非0对象】

BigInteger的存储方式

将被存储数的补码每32位为一组，转为10进制存入数组mag[]

如：

68719476733-》1111 1111 1111 1111 1111 1111 1111 1111 1101

后32位转为补码为-》1000 0000 0000 0000 0000 0000 0000 0011

转为十进制-》-3

前4位为正，补码与原码相同-》0000 0000 0000 0000 0000 0000 0000 1111

转为十进制-》15

所以mag = [15,-3]

舍入模式使用了一个枚举类RoundingMode包装

RoundingMode 是 Java 中 BigDecimal 类使用的一个枚举类型，用于指定在进行四舍五入操作时的舍入模式。

  1. UP：远离零方向舍入，正数舍入为更大的数，负数舍入为更小的数
  2. DOWN：靠近零方向舍入，正数舍入为更小的数，负数舍入为更大的数
  3. CEILING：向正无穷方向舍入，正数舍入为更大的数，负数舍入为更小的数（对正数等同于 UP，对负数等同于 DOWN）
  4. FLOOR：向负无穷方向舍入，正数舍入为更小的数，负数舍入为更大的数（对正数等同于 DOWN，对负数等同于 UP）
  5. HALF_UP：四舍六入五取偶，即当舍入位的后一位是5时，如果舍入位是奇数，则进位；如果是偶数，则舍去
  6. HALF_DOWN：四舍六入五也入，即当舍入位的后一位是5时，舍入位无论奇偶都舍去
  7. HALF_EVEN：银行家舍入法，即当舍入位的后一位是5时，如果舍入位是奇数，则进位；如果是偶数，则舍去。这与 HALF_UP 行为类似，但当数字刚好在中间值时，它将数字舍入到最近的偶数
