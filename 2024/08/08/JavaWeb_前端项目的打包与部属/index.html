前端项目的打包与部署

打包

对于Vue项目，一条指令即可

npm run build

打包完成后，项目中的dist文件夹就是打包好的前端文件

部署

Nginx：轻量级Web服务器

Web服务器：处理HTTP请求并提供网页内容

nginx安装目录介绍

  • conf：配置文件目录

  • html：静态资源文件目录

  • logs：日志文件目录

  • temp：临时文件目录

将打包好的前端文件放入html文件夹

双击exe启动服务器

可能出现的错误：

端口号被占用，nginx使用的默认端口号为80

可以通过cmd查询端口信息

返回信息表示进程号62636正在占用此端口，可以去任务管理器中查看该进程号

也可以输入指令找出使用该端口的具体应用程序

（当然，因为我没有出现错误，所以这个进程就是nginx.exe）

关于返回信息的含义

  • TCP：表示这是一个TCP协议的连接。
  • 0.0.0.0:80：表示监听在所有网络接口的80端口上。0.0.0.0 是一个通配符地址，表示监听在所有可用的网络接口上。
  • 0.0.0.0:0：表示没有特定的远程地址与之连接，因为这是一个监听状态。
  • LISTENING：表示这个端口正在等待连接请求，处于监听状态。
  • 62636：是占用端口80的进程的进程ID（PID）。

如果想更改nginx的默认端口号，可以修改conf下的nginx.conf中的listen项

nginx功能强大，需要进一步学习
