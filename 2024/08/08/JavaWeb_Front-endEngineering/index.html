前端工程化

在学习前端工程化之前，先了解一些小知识

  • 关于开发模式

  • 关于Api

开发模式

混合开发

在早期，采用的是前后端混合开发，需要后端人员处理前端人员编写好的前端页面，这样的开发模式有以下几个问题

沟通成本高：后端人员发现前端代码出现问题，需要沟通。

分工不明确：后端人员需要操作前端代码

不便管理&不便维护和扩展：前后端代码混合，无论是修改or添加需求都需要阅读大量代码

分离开发

鉴于混合开发的问题，当前主流的的开发模式为前后端分离开发。

前端工程负责页面构建，后端工程负责数据处理，前后端对接需要某些开发规范（前端向什么地方发送请求？后端如何向前端反馈数据？），这个开发规范就是接口，接口文档里会记录所有需要的业务功能，一个项目的前后端工程需要根据这份文档进行开发

神奇文档从何而来

接口文档通过项目原型+市场需求分析得来，出自项目经理

项目开发流程

  1. 需求分析

  2. 接口定义

  3. 前后端并行开发

  4. 独立测试

  5. 前后端联调测试

什么是前端工程化

在企业级的前端项目开发中，把前端开发所需的工具、技术、流程、经验等进行规范化，标准化。前端工程化涉及四个关键点：

模块化：将JS、CSS制作成可复用的模块

组件化：将UI结构、JS、CSS封装为组件

规范化：提供标准的目录结构、编码、接口

自动化：项目的构建、部署、测试可以自动化的进行

人为处理这四个关键点效率不高，因此我们需要一个工具

Vue-cli（Vue的脚手架）

Vue-cli是什么

Vue官方提供的一个脚手架工具，用于快速生成一个Vue项目模板

Vue-cli具有以下功能：

  • 统一的目录结构

  • 本地调试

  • 热部署

  • 单元测试

  • 集成打包上线

依赖环境：Node.js — Run JavaScript Everywhere (nodejs.org)（一个前端工程化(JS)环境）

配置环境变量(需要管理员权限)

npm config set prefix "nodeJS文件夹的绝对路径"

使用以下命令检查版本

node -v

切换淘宝镜像

npm config set registry https://registry.npmmirror.com

查看镜像使用状态

npm config get registry

安装vue-cli

npm install -g @vue/cli

目录结构

基于Vue脚手架创建的工程，有标准的目录结构：

node_modules

项目依赖包

public

项目静态文件

src

项目源码

作为最常用的工作区间，需要了解其内部结构

  • assert：静态资源

  • components：可重用组件

  • router：路由配置

  • views：视图组件（页面）

  • App.vue：入口页面（根组件）

  • main.js：入口js文件

重要文件

package.json

项目基本信息、所需资源信息，资源版本信息

vue.config.js

vue项目的配置信息

启动项目

npm run serve
