关于网络

家里网很卡，想修一下却发现自己啥都不懂。。。

记录一些名词和数据基本的传输过程

IP地址

主机在网络中的位置，网络位和主机位共同构成了完整的IP地址

子网掩码

子网掩码用来确定IP地址中的网络位和主机位。子网掩码中的”1”表示网络位，”0”表示主机位。当访问某ip时会根据子网掩码判断是否处于同一网络位，以此决定是否需要网关转发

网络位：ip地址前几位数字，代表主机所在网段，这些数字叫网络位，同网段间的消息不用通过网关转发

例：

IP:192.168.1.1

子网掩码:255.255.255.0

表示192.168.1是网络位

这里的255是8位二进制1转化得到的，实际为11111111.11111111.11111111.00000000

一般某段会为255或0，特殊情况如255.255.255.252，转为2进制为11.11.1~1.11111100，表示前30位是网络位，后两位为主机位，只要一段连续的1接着一段连续的0就是合法的。

网关

用于不同网段的数据转发与接收

DNS

域名系统，它将人类易于记忆的域名（如 www.example.com）转换为机器使用的IP地址（如 192.168.1.1）。这样，用户在浏览器中输入域名时，DNS 就能够将域名解析为相应的IP地址，使得用户能够访问到目标网站。

DNS 系统由多个层次的 DNS 服务器组成，包括根服务器、顶级域服务器、权威名称服务器和本地DNS服务器。当用户查询域名时，查询请求会从本地DNS服务器开始，逐级向上查询，直到找到对应的IP地址。

关于DNS服务器组成

  1. 根 DNS 服务器（Root DNS Servers）
  • 根服务器是 DNS 架构的最顶层，它们包含对顶级域（TLD）的引用。全球有 13 组根服务器，每组服务器都以字母 A 到 M 命名。
  • 当一个 DNS 查询开始时，它首先到达根服务器，根服务器会告诉查询者下一步应该查询哪个顶级域服务器。
  1. 顶级域（TLD）服务器（Top-Level Domain Servers）
  • 顶级域服务器负责管理顶级域名，比如 .com、.org、.net 或国家代码顶级域名（ccTLDs），如 .uk、.cn。
  • 当根服务器将查询指向相应的 TLD 服务器后，TLD 服务器会提供负责特定二级域名的权威名称服务器的信息。
  1. 权威名称服务器（Authoritative Name Servers）
  • 这些服务器包含了特定域名的 DNS 记录，是对该域名进行解析的最终权威来源。
  • 它们存储了域名到 IP 地址的映射，以及可能的其他类型的 DNS 记录，如 MX 记录（用于邮件服务器）或 CNAME 记录（用于别名）。
  1. 本地 DNS 服务器（Local DNS Servers）
  • 本地 DNS 服务器通常是互联网服务提供商（ISP）提供的，它们是用户设备用来进行 DNS 查询的第一站。
  • 当用户的设备发起 DNS 查询时，首先会询问本地 DNS 服务器。如果本地 DNS 服务器缓存了答案，它会直接返回结果；如果没有，它会代表用户向上级 DNS 服务器发起查询。
  1. 递归 DNS 服务器（Recursive DNS Servers）
  • 递归服务器是执行递归查询的 DNS 服务器，它们在收到查询请求后，会连续查询多个 DNS 服务器直到找到答案。
  • 本地 DNS 服务器通常也作为递归服务器，代表用户完成整个查询过程。
  1. 缓存 DNS 服务器（Caching DNS Servers）
  • 缓存服务器存储了之前查询过的结果，以减少对权威服务器的查询次数，加快解析速度。
  • 本地 DNS 服务器通常具有缓存功能，它们会保存一段时间内的 DNS 查询结果。
  1. 任何铸服务器（Anycast DNS Servers）
  • 任何铸是一种将一个 IP 地址分配给多个服务器的技术，使得客户端可以连接到最近的服务器。
  • 某些 DNS 服务提供商使用任何铸技术来提供 DNS 服务，以提高性能和冗余。

关于查询流程

  1. 客户端发起查询：
    用户在浏览器中输入一个网址，浏览器会生成一个 DNS 查询请求，以获取该域名对应的 IP 地址。

  2. 本地 DNS 服务器查询：
    查询首先发送到用户的本地 DNS 服务器（通常是用户的互联网服务提供商 ISP 提供的）。如果本地 DNS 服务器缓存了该域名的 IP 地址，它将直接返回结果。

  3. 递归查询：
    如果本地 DNS 服务器没有缓存结果，它将代表客户端向上级 DNS 服务器发起查询。这个过程称为递归查询。

  4. 根服务器查询：
    如果本地 DNS 服务器不知道答案，它会联系根 DNS 服务器。根服务器不会提供最终的 IP 地址，但会告诉本地 DNS 服务器应该查询哪个顶级域（TLD）服务器。

  5. 顶级域服务器查询：
    根据根服务器的指引，本地 DNS 服务器接着联系相应的顶级域服务器（例如 .com、.org 等）。顶级域服务器会提供负责该域名的权威名称服务器的信息。

  6. 权威名称服务器查询：
    然后，本地 DNS 服务器联系权威名称服务器，这些服务器包含了域名的最终 DNS 记录。权威名称服务器会返回域名对应的 IP 地址。

  7. 返回结果：
    一旦本地 DNS 服务器获得了 IP 地址，它会将这个信息返回给客户端，并可能将其缓存起来，以便将来的查询可以直接使用缓存的结果。

  8. 客户端连接：
    客户端（如用户的浏览器）收到 IP 地址后，会使用该地址尝试连接到目标服务器，完成整个 DNS 查询过程。
