|Post Views:

内部类

写在类里的类就是内部类

什么时候用到内部类

内部类属于类的一部分，单独存在无意义。

内部类可以直接访问外部类的成员，包括私有

外部类要访问内部类的的成员，需先创建内部类对象

内部类的分类

成员内部类

静态内部类

局部内部类

匿名内部类

成员内部类

写在成员位置的，属于外部类的成员

获取成员内部类对象的两种方式

当成员内部类非私有时，可以直接创建对象

Outer.Inner oi = new Outer.new Inner();

当成员内部类被private修饰时，需要在外部类编写方法，对外提供内部类对象

  • 使用多态，用父类object接受

  • 直接使用对象

关于类中不同区域的变量如何访问

内部类中方法访问

  • 方法内变量：变量名

  • 内部类重名变量：this.方法名

  • 外部类重名变量：外部类名.内部类名.变量名

在内存中，内部类会存储外部类地址值，存于变量(外部类名.this)中

关于静态成员变量与方法

注意：在成员内部类中（非静态）定义静态的变量及方法，在JDK16以前是不被允许的。JDK16及以后允许在成员内部类中定义静态的变量与方法

静态内部类

静态内部类时一种特殊的成员内部类

直接创建静态内部类对象的方式

外部类名.内部类名 oi = new 外部类名.内部类名();

如何调用静态内部类中的方法

非静态方法：先创建对象，用对象调用

静态方法：外部类名.内部类名.方法名()

例：类Outer中有内部类Inner，Inner中有普通和静态方法

静态方法可以通过外部类.内部类.方法使用，而普通方法需要创建内部类对象，通过对象调用。

局部内部类

将内部类定义在方法里面就叫做局部内部类，类似于方法里面的局部变量。
外界是无法直接使用，需要在方法内部创建对象并使用。
该类可以直接访问外部类的成员，也可以访问方法内的局部变量

匿名内部类

隐藏名字（虚拟机自动添加）的内部类，可以写在成员位置（一种成员内部类），也可以写在局部位置（一种局部内部类）。

格式与理解

class Dog extends Animal

dog类继承Animal类

这个内部类的意思为：创建某个类（狗）的对象继承了类Animal，创建对象（new）时调用某个类（狗）的空参构造。

细节

包含了继承或实现，方法重写，创建对象。

  • 继承或实现：类名或接口名

  • 方法重写：{重写方法}

  • 创建对象：new

使用场景

当方法的参数是接口或者类时，

以接口为例，可以传递这个接口的实现类对象，

如果实现类只要使用一次，就可以用匿名内部类简化代码。

例：抽象类Animal

此时调用方法text，需要的参数为抽象类Animal的实现类（多态），此时可以直接使用匿名内部类，直接传入一个【无名的（匿名）】【Animal实现类】的对象，而不用先创建实现类，再实例化对象，最后传入对象以使用方法。因此，匿名内部类适合只需使用一次的实现类 （不用单独创建一个对象（带名字的））

当然接口也可以实现

使用接口Swim接受其实现类：接口多态
