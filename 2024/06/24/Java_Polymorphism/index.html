Created2024-06-24|Updated2025-07-12|Java
|Post Views:

多态

对象的多种形态

面向对象的三大特征

封装

继承

多态

将零散的属性和方法按特定规则组合在一起，是封装

封装后成为类

两种类具有相似的性质，将相似部分提取出来作为它们的父类，是继承

继承使类与类之间产生联系，通过继承，我们可以从一个类延申出多种子类，这些更为细化的子类在成为一种新类的同时，同样属于其父类的类，也就是一个类可以属于多个类（多层继承），这就是多态

多态使用的前提

有继承/实现关系

有父类引用指向子类对象

有方法的重写

多态的应用场景

用父类类型接收子类对象

例：Student和Teacher继承自Employer类，并重写了Employer的print方法，login方法会调用Employer类的print方法，此时Employer的子类同样可以传入该方法，并且会优先使用自己类中的方法。

子类重写了print方法

多态的好处

多态模式下，右边对象可以实现解耦合，提升了可维护性

扩展性

多态调用成员的特点

Animal animal = new dog();

变量调用：编译看左边，运行也看左边

方法调用：编译看左边，运行看右边

编译看左边：

虚拟机的检查原则，多态调用成员时，会检查父类是否有这个成员，如没有则会报错。

运行看左/右边：

程序运行时，成员变量看左边，即获取父类的值。

程序运行时，成员方法看右边，即调用子类的方法。

为什么？

成员变量在继承中不会被覆盖，而统计方法的虚方法表会随子类重写父类的方法而覆盖原方法。

重现一下使用多态时的场景：

Animal a = new Dog();

a是引用类型变量，存一个Animal类对象的地址

BUT

使用new对象开辟一块内存并调用了Dog类的构造方法，因为Dog类是Animal类的子类，因此其地址可以用Animal类引用变量存储（多态），而Dog类中如果重写了父类的方法，那Dog类的虚方法表中的这个方法将被修改，即没有Animal的这个方法，则创建出的对象Dog拥有的是重写过后的（Dog类）方法。但变量不会被重写，所以这个变量属于父类，调用时调用的就是父类的变量（给某个子类对象的某个父类拥有的成员变量赋值时，这个变量就是父类的，不用在子类中再写一遍）。

由此得出多态的一个问题：使用多态时，我们无法调用子类特有的部分（这里的特有是相较于父类特有，即父类没有这个成员，是子类新加的成员），如需调用，需要进行类型的转换，即转为子类类型。

需要避免转换成其他子类

使用instanceof关键字

instanceof：判断某个对象是否为某个类
