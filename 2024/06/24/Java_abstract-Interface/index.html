abstract&接口

abstract关键字

抽象（果真吗？）

未知实现方式但是已经确定存在的部分，就是抽象

抽象方法

将共性的行为(方法)抽取到父类之后。由于每一个子类执行的内容是不一样的，所以在父类中不能确定具体的方法体时，该方法就可以定义为抽象方法。

比如现在有个动物类Animal，已知需要实现方法eating()表示动物吃的食物，其子类所代表的每种动物吃的都不一样，所以需要子类补充该方法的实现（方法体）。

抽象类

存在抽象方法的类

注意事项：

  • 抽象类不能实例化

  • 抽象类可以有构造方法（确实抽象）

  • 抽象类中不一定有抽象方法，有抽象方法的一定是抽象类。

  • 抽象类的子类需实现其继承的所有抽象方法，否则声明为抽象类

为什么不能实例化，还会有构造方法？

其子类在调用它自己的构造方法时，会首先调用其父类的构造器(super())，以创建父类中的成员，如果这个父类是抽象类，这里就会用到构造方法，即【抽象类的构造方法】将用于其【非抽象的子类】的构造。当然，因为本身残缺（含未实现的抽象方法），不能实例化对象。

Interface

接口

向一个类添加一个功能？

可以说接口是对java仅单继承的一种解决办法，但更多时候用于规范格式。

接口不是一种类，这从它的”继承“方式implement（实现）（这里强调接口用的不是类的extends)，也可以看出，它比较特殊。

那么问题来了，接口与抽象类有什么不同？语法上不同点不少，但逻辑功能上呢？

这有一个很好的区别方式：类的继承有一种很强的定义关系，即子类是父类的一种，称为is-a关系；而接口的作用是实现功能。一个类实现某个接口后，并不会成为那个接口——人遵守规则，但人不是规则；类实现接口（规范），但类不是接口。

使用方式

  • 用关键字interface来定义

public interface 接口名{}

  • 不能实例化

  • 接口与类之间是实现关系，通过implements关键字表示

public class 类名 implements 接口名{}

  • 接口的子类(实现类)需重写接口中的所有抽象方法
    或声明为抽象类

接口中的成员特点

成员变量

只能是常量

默认修饰符：public static final

默认：即使不写，虚拟机也会自动添加

public：公共的

static：静态方法，方便调用

final：常量，接口中是设定的规则，不可改变

构造方法

接口中没有构造方法

成员方法

实现类根据具体需求实现

默认修饰符：public abstract

JDK7以前只能为抽象方法

JDK8:接口中可以定义有方法体的方法（默认default或静态static）

在解决接口升级（如接口方法增加）问题时，为保证兼容性（接口要求实子类实现全部抽象方法，接口抽象方法增加后非抽象子类都需实现，这会导致原来的代码无法编译）而出现新特性。

JDK9:接口中可以定义私有方法

因为接口中可以实现具体的方法，这些方法之间可能会出现重复使用的代码，而没有私有方法之前，整合出包含重复代码的方法可以被外界调用，这是没有意义的。对此，在JDK9中提出：接口可以定义私有方法。

新特性

默认（default）方法

接口中的默认方法具有方法体，不是抽象方法，因此不强制重写

如果需要重写，重写时去掉default关键字

如果一个类实现了多个接口，这些接口中存在名字相同的默认方法，那这个类必须重写该重名方法（否则不知道此方法来自哪个接口）

当然，在接口多继承时，也要重写重名的默认方法（猜的）

静态（static）方法

同样是一个具有方法体的方法，但是从属于接口，只能通过接口调用，不能通过实现类名或者对象名调用，也不能被重写

私有（private）方法

在接口中，私有方法的产生是为了提高其中已实现（默认、静态）方法的复用性

普通的私有方法——服务于默认方法

但静态方法中只能也使用静态方法，因此

静态的私有方法——服务于静态方法

可以说私有方法的类型（普通or静态），基本可以根据其调用者来选择。（虽然普通方法也能调用静态的私有方法）

例：a、b是默认方法，c、d是静态方法，print()是普通的私有方法，printStatic()是静态的私有方法。

接口与类的关系

类与类：继承关系

只能单继承，不能多继承，但能多层继承

类与接口：实现关系

接口和类的实现关系，可以单实现，也可以多实现 public class 类名 implements 接口名1，接口名2 {}

由继承得来的子类同样可以实现接口

`public class 类名 extends 父类 implements 接口名1，接口名2 {}

注意：当类多实现时，需要实现接口中的所有抽象方法，否则声明为抽象类

接口与接口：继承关系

可以单继承，也可以多继承

注意：当类实现的接口为继承了其他接口的子接口时，需要实现此子接口体系（它与它继承的接口）中所有的抽象方法

例：当前接口Inter3继承了接口Inter1和Inter2，此时创建实现类InterImpl实现Inter3

接口的多继承

实现类需要实现接口体系的所有抽象方法

适配器设计模式

设计模式

一种设计套路

适配器设计模式

当一个接口中抽象方法过多，但只要使用其中一部分的时候，就可以适配器设计模式。用一个中间类空实现接口，再让实现类继承中间类并重写所需方法。

注意：这个中间类是无意义的，因此使用abstract修饰，避免创建中间类对象。

例：接口Inter1有五个抽象方法，我只想用两个

可以不用再实现类里加这么多空实现了。
