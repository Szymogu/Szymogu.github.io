static修饰

修饰成员变量

从属于类，先于对象生成，所有对象共用

修饰成员方法

多用于测试类和工具类中

工具类：一些方法的集合，不能创建对象实例

例：定义了工具类ArrUtil,提供了两数相加，相减的方法。

注意事项

  • 静态方法只能访问静态变量和静态方法（静态方法中只访问静态）

  • 非静态方法可以访问静态变量或者静态方法，也可以访问非静态的成员变量和方法（非静态可以访问所有）

  • 静态方法中没有this关键字（静态中没有this关键字）

此外，static在单例模式中也有运用

有趣的理解：与对象无关

静态方法的加载时机

静态方法随类加载，而类的加载有以下几种场景：

  1. 在新创建一个对象时：

    • 当使用new关键字创建一个类的实例时，如果该类还没有被加载，则需要先加载该类。

  2. 在访问类的静态成员时：

    • 当访问一个类的静态变量或者调用静态方法时，如果该类还没有被加载，则需要先加载该类。

  3. 在反射时：

    • 当使用Java反射API对类进行操作时，如Class.forName的调用，如果该类还没有被加载，则需要先加载该类。

  4. 在初始化一个类时：

    • 当某个类在初始化其父类时，如果父类还没有被加载，则需要先加载父类。

  5. 在调用接口中的常量时：

    • 接口中的常量（public static final）在接口被加载时就已经被初始化，不需要显式加载接口。

  6. 当JVM启动时：

    • 启动类（包含main方法的类）会被加载。
  • 单例模式
